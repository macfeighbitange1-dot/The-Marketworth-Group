Orchestrating the Swarm

[ COURSE: AGENTIC SWARMS // MODULE 02 // AGENT ROLE-PLAY ]

In Lesson 1, we gave the AI a brain. In Lesson 2, we give the AI Identity and Purpose. An agent is simply an LLM wrapped in a "Persona" with specific goals and tools.

01. The Environment Setup

We need to install the orchestration layer. We use CrewAI because it handles "Process Flow"—ensuring Agent A finishes before Agent B starts.

pip install crewai langchain_community
02. Connecting the Local Brain

By default, these frameworks look for OpenAI. We must force them to look at your Local Ollama Instance (running on port 11434).

from langchain_community.llms import Ollama

# Initialize your local brain
local_llama = Ollama(model="llama3.1")
03. Defining the Personas

We will create two agents: a Researcher (to find data) and a Writer (to format it). This is a "Sequential Flow."

from crewai import Agent, Task, Crew

# Agent 1: The Market Analyst
analyst = Agent(
  role='Market Researcher',
  goal='Find the top 3 AI trends in Kenya for 2026',
  backstory='Expert in East African tech emerging markets.',
  llm=local_llama
)

# Agent 2: The Content Architect
writer = Agent(
  role='Senior Tech Editor',
  goal='Write a viral LinkedIn post about the research',
  backstory='Specialist in high-conversion tech copywriting.',
  llm=local_llama
)
04. Executing the Mission

We combine them into a "Crew." This is where the 0xCC2 magic happens: the agents collaborate without your help.

crew = Crew(
  agents=[analyst, writer],
  tasks=[task1, task2], # (Tasks defined in full code sample)
  verbose=True
)

result = crew.kickoff()
print(result)

Step-by-Step Procedure Summary:

  1. Start Ollama in your terminal: ollama serve
  2. Write your Python script defining Agents (Who they are) and Tasks (What they do).
  3. Connect them via the Crew object.
  4. Run python swarm_01.py and watch the terminal logs as they "talk" to each other.
